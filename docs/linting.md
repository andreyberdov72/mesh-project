# Лінтинг у проєкті

## Використовувані інструменти

- **flake8** – базовий лінтер для перевірки стилю та логічних помилок.
- **pylint** – глибокий аналіз коду (іменування, складність, дублювання).
- **bandit** – перевірка безпеки (виявлення потенційно небезпечних конструкцій).

## Причини вибору

- **flake8** швидкий і простий у налаштуванні, покриває основні правила PEP8.
- **pylint** надає розширені перевірки, які допомагають підтримувати високу якість коду.
- **bandit** забезпечує додатковий рівень безпеки, що важливо для скриптів, які працюють з системними викликами.

## Налаштування

Конфігураційні файли знаходяться в корені проєкту:

- `.flake8`
- `.pylintrc`
- `.bandit`

Основні правила:
- Максимальна довжина рядка: **100 символів**.
- Ігноруються директорії: `myvenv`, `bin`, `tftp_temp`, `keys`, `tftp-now`, `openwrt-image-builder`, `MeshGraphViewer`, `meshnet-lab`
- Вимкнені деякі повідомлення pylint (докстрінги, надто мало публічних методів) для зменшення шуму.

## Запуск лінтерів

### Локально (вручну)

```bash
# flake8
flake8 .

# pylint
pylint *.py

# bandit
bandit -r .
