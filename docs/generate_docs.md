# Як генерувати документацію

Цей проєкт використовує **Sphinx** для генерації документації з вихідного коду (завдяки рядкам документації) та Markdown/reStructuredText файлів. Документація підтримує багатомовність (українська та англійська).

## Необхідні інструменти
Для локальної генерації документації необхідно встановити залежності:
```bash
pip install -r docs/requirements.txt
```

## Генерація HTML-документації
Для того, щоб зібрати документацію локально, скористайтеся інструментом `make` всередині папки `docs/`.

### 1. Збірка українською мовою (Основна)
```bash
cd docs
make html
```
Результат буде збережено у папці `docs/_build/html/ua/`. Вхідним файлом слугуватиме `index.html`.

### 2. Збірка англійською мовою
```bash
cd docs
make en
```
Результат буде збережено у папці `docs/_build/html/en/`. Вхідним файлом слугуватиме `index.html`.

## Перевірка якості (Лінтинг документації)
У цьому проєкті налаштовано сувору перевірку якості документації за допомогою `pydocstyle` та `Sphinx` (з режимом "warnings as errors"). 

Щоб вручну перевірити покриття коду документацією:
```bash
pydocstyle openwrt-generator/ bulk-flasher/
```

## Жива документація (Test-Driven Documentation)
Завдяки вбудованим `doctest` тестам у Python-коді, ви можете автоматично перевіряти, чи є приклади у документації робочими.
Для цього виконайте:
```bash
cd docs
make doctest
```
Це виконає всі приклади коду з документації як тести і повідомить, якщо вони застаріли.
